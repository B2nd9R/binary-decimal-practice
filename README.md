# Binary to Decimal Practice (Arabic RTL)

تطبيق ويب تدريبي تفاعلي يساعد الطالب على إتقان التحويل من النظام الثنائي (Binary) إلى النظام العشري (Decimal) خطوة بخطوة، بدل الاكتفاء بمحوّل مباشر.

## لماذا هذا المشروع؟

الفكرة التعليمية هنا أن المتدرب يفهم المنطق، وليس فقط النتيجة:

- كل خانة في العدد الثنائي لها وزن مكاني من قوى 2.
- الأوزان تبدأ من اليمين عند 2^0 وتزداد كلما اتجهنا لليسار.
- الناتج العشري يساوي مجموع القيم الناتجة من الخانات التي فيها 1.

## أهم الميزات

- توليد رقم ثنائي عشوائي بطول من 4 إلى 8 بت.
- إمكانية اختيار الطول: عشوائي أو طول ثابت (4 إلى 8).
- عرض العدد الثنائي بشكل واضح في أعلى الواجهة.
- الخطوة 2 تلقائيًا: عرض البتات مع قوى 2 المناظرة.
- الخطوة 3 تلقائيًا: عرض الصيغة مثل:
  1×2^5 + 0×2^4 + 1×2^3 + ...
- ترك الخطوة 1 والخطوة 4 والخطوة 6 للمتدرب للكتابة اليدوية.
- زر التحقق من الإجابة مع تغذية راجعة فورية.
- زر إظهار الحل الكامل (التفصيل + مجموع الخانات المفعلة فقط).
- زر تلميح يعطي نقطة بداية ذكية للحل.
- لوحة إحصائيات مباشرة:
  - عدد المحاولات
  - عدد الإجابات الصحيحة
  - السلسلة الحالية
  - نسبة الدقة
- واجهة عربية RTL ومتجاوبة للجوال وسطح المكتب.
- خط عربي أساسي IBM Plex Sans Arabic وخط رقمي قوي للأرقام.

## صور ذهنية للتدفق التعليمي

1. توليد سؤال Binary جديد.
2. المتدرب يكتب تصور قوى 2 (الخطوة 1).
3. النظام يعرض الربط التلقائي للبتات مع القوى (الخطوتان 2 و3).
4. المتدرب يكتب التقييم العددي (الخطوة 4).
5. المتدرب يدخل الناتج النهائي بالعشري (الخطوة 6).
6. النظام يتحقق ويعرض نجاح/خطأ مع إمكانية عرض الحل الكامل.

اقتراح أفضل أثناء التطوير:

- استخدام إضافة Live Server لتحديث الصفحة تلقائيًا عند الحفظ.

## منطق التحقق الرياضي

- يتم حساب القيمة الصحيحة داخليًا عبر تحويل الثنائي إلى عشري.
- في JavaScript:
  parseInt(binaryValue, 2)
- ثم تتم مقارنة الناتج بإجابة المستخدم النهائية.

## قابلية التطوير مستقبلًا

- وضع اختبار زمني بعدد أسئلة محدد.
- نظام مستويات (سهل/متوسط/صعب) مع شروط نجاح.
- حفظ التقدم محليًا عبر Local Storage.
- صفحة نتائج نهائية مع تحليل نقاط الضعف.
- دعم التحويل العكسي من Decimal إلى Binary.
